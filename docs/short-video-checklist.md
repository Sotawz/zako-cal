# ショート動画チェックリスト

Phase 6では、スマホ実機で画面録画し、需要検証に使えるかを確認します。最初の動画はfallback推定で十分です。

## 画面録画前

- VercelのデモURLをスマホで開ける。
- 公開デモでは `OPENAI_API_KEY` を設定していない。
- `?demo=1` を付けたURLで、結果トーク画面のタブが非表示になる。
- アップロード用に分かりやすい食事写真を用意している。

## 動画内で見るポイント

- 冒頭1秒で「何のアプリか」が分かるか。
- 食事画像が見えるか。
- カロリー数字が大きく見えるか。
- キャロリンの顔が見えるか。
- 煽り文が1〜2行で読めるか。
- 画面録画でUIボタンが邪魔になっていないか。
- 「使ってみたい」と思えるCTAがあるか。
- トップから結果までが3〜7秒で伝わるか。

## 推奨フロー

1. トップ画面を開く。
2. 「食事を記録する」を押す。
3. 食事写真を選ぶ。
4. 「カロリーを計測する」を押す。
5. 解析中画面を短く見せる。
6. キャロリントーク画面で止める。

## 動画タイトル案

- 食べた飯をAIに見せたら煽られた
- 二郎系ラーメンをカロリーAIに見せた結果
- カロリー管理アプリなのに口悪すぎた

## NGチェック

- コメントが3行以上になっている。
- キャロリンの顔にテキストが被っている。
- カロリー数字が小さすぎる。
- アップロード前後の操作が長く、動画のテンポが落ちている。
- 健康管理アプリとしての安心感より、ネタ感だけが強く見えている。
