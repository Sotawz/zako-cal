# キャロリン モーション設計メモ

## 現在の状態

- Phase 2では `public/samples/calorin.png` をキャロリンの仮素材として使用する。
- アプリ内の正式表示名は「キャロリン」。ファイル名は現状の `calorin` のまま扱う。
- 表示箇所はAIコメント欄と解析中画面を中心にし、アプリ全体は白基調の食事記録UIを維持する。
- 実装は `CarolynAvatar` コンポーネントに分離し、同じPNGをCSSのトリミングと軽いアニメーションで使い回す。

## Phase 2のモーション方針

- 本物の3D、Live2D SDK、音声生成はまだ導入しない。
- CSSアニメーションのみで、軽い上下揺れ、ポップ表示、hover時の小さな傾きに留める。
- `animated` propsで演出をON/OFFできるようにし、将来の差し替えポイントを明確にする。
- 解析中は小さく揺れる程度、結果画面ではコメントカードがふわっと出る程度にする。

## Phase 2.5の表示方針

- 既存のPNGキャラ画像を使い、AIコメント欄だけVTuber風に見せる。
- `CarolynAvatar` は `face`、`bust`、`full` の表示モードを持ち、丸アイコンと全身/腰上風の表示を使い分ける。
- `CarolynStage` を追加し、キャロリンがコメント横で喋っている構図を作る。
- `CarolynStage` は将来のLive2D/3D差し替えポイントにする。
- 短期的にはCSSアニメーションで上下揺れ、ポップ表示、hover時の小さな傾きだけを使う。
- UI全体は白基調の食事記録アプリとして保ち、キャラクター感はAIコメント欄と解析中画面に限定する。

## Phase 4の表示修正方針

- キャロリンは顔と表情が見えることを最優先にする。
- 全身表示で顔が見切れる場合は、`bust` 表示へフォールバックする。
- AIコメント欄、シェア用表示、解析中画面では `CarolynStage` 内でバストアップ表示を使う。
- 「このキャラが喋っている」とスクショで一瞬で分かる状態を優先し、完璧な全身切り抜きにはこだわらない。

## Phase 4.5のトーク表示方針

- `CarolynTalkScene` を追加し、通常結果画面とは別に9:16のショート動画向け表示を作る。
- トーク表示ではキャロリンを画面中央〜下部に大きく配置し、AIコメントをチャット吹き出しとして重ねる。
- 食事名、推定カロリー、今日の合計は小さなチップに留める。
- 将来的にLive2D/3Dへ進む場合、`CarolynTalkScene` 内の大きなキャロリン表示を差し替えポイントにする。

## Phase 4.6の作り直し方針

- トーク表示ではキャロリンが画面の40〜70%を占める構成にする。
- 通常の食事記録カードUIとは明確に分け、食事情報はチップ程度に抑える。
- 顔、表情、上半身が必ず見えるよう、`displayMode="talk"` の専用クロップを使う。
- 吹き出しは顔を隠さず、胸元付近に重ねる。

## Phase 4.7の判定フロー方針

- サンプル選択後は「カロリーを計測する」から解析中画面を経由し、最初に全画面キャロリン判定を表示する。
- 通常結果カードは「詳細」表示として残し、バズ用の主画面は全画面キャロリンにする。
- トーク表示ではキャロリンが画面の60〜80%を占めることを目標にする。
- 推定カロリーと煽りコメントをキャロリンの上に重ね、食事情報は小さなチップへ抑える。

## 将来候補

### Live2D Cubism

- 2Dキャラを自然に動かせるため、キャロリンのようなアプリマスコットと相性が良い。
- 表情、口パク、軽い身体の揺れを追加しやすい。
- 中期的な本命候補。需要検証後、キャラ表現を強める段階で検討する。

### Rive / Lottie

- 軽量なUIアニメーション向き。
- 表情差分、吹き出し、リアクション演出などに向いている。
- Live2Dより簡単に導入できるが、キャラの自由度は素材設計に依存する。

### VRM + three.js / react-three-fiber

- 本物の3Dキャラクター表示に向いている。
- 視線、姿勢、カメラ演出などを作れる一方で、実装と表示負荷が重くなりやすい。
- 3D化するならVRM + three.js / react-three-fiberを候補にする。
- 本格3Dは重くなりやすいため、需要検証後に判断する。

## 次に検証したいこと

- AIコメント欄にキャロリンがいることで、コメント体験が分かりやすくなるか。
- キャラを出しすぎず、普通の食事記録アプリ感とのギャップが成立するか。
- ショート動画で、キャロリンの登場が一瞬で伝わるか。
- 将来音声化したときに、コメントカードとキャラ表示が自然につながるか。
